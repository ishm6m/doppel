# TASKS.md — Implementation Plan

> **Purpose:** An ordered, checkable task list. Claude Code picks the next unchecked task, implements it (with tests), meets its DoD, then moves on.
> **Scope:** MVP through V1, broken into milestones. Each task ≈ one focused PR.
> **Dependencies:** All specs. Tasks reference FR-/F-IDs.

**Working rule:** Do tasks top-to-bottom. Don't start a task until the previous one's DoD (code + tests + lint) passes. Mark `[x]` when done.

## Milestone 3 — Engine: Stage 2 (the headline)
- [x] **T3.1** Text extractors: txt/md (`PlainTextExtractor`) + **`.docx`** (`DocxTextExtractor`). DoD met: .docx is read with a pure-Swift central-directory ZIP parser + Foundation `Compression` (raw DEFLATE) and OOXML→text stripping — no dependency, no AppKit, engine stays pure. Wired into the coordinator + compare dispatch; covered by `DocxExtractTests` (real base64 .docx fixture → exact normalized text, corrupt-blob → no text, run/entity handling). **RTF still deferred** (NSAttributedString is AppKit-only and would taint the engine); revisit behind a host-side extractor if needed.
- [x] **T3.2** PDF extractor (PDFKit text layer) + scanned-PDF classification (F5). DoD met: `PDFTextExtractor` classifies text-layer vs scanned (sparse-layer heuristic); text-PDFs routed into Stage 2 near-dup, scanned PDFs surfaced as `.needsOCR` skips (never dropped). Vision OCR pass itself is opt-in, deferred (F5 "Run OCR" / T7-era).
- [x] **T3.3** Shingling + MinHash (128 perms) + LSH banding (16×8) + Jaccard estimate. DoD met: near-dup pair detected; threshold via `DetectionConfig`.
- [x] **T3.4** Near-dup grouping + `reasonSummary` via fast diff. DoD met: precision 1.0 on constructed near-dup set (≥0.95); LSH pruning asserted.

- [x] **T5.1** Text diff engine (word) + compare view (F8). DoD met: `TextDiff` (stdlib CollectionDifference) flags changed words per side; `CompareView` renders two panes with changed words highlighted; "Compare" affordance on each non-keeper row diffs it against the keeper. Contract-date demo proven by `testTextDiffHighlightsOnlyTheChangedWord`. (Diffs the *normalized* text the engine matched on; original-formatting compare deferred.)
- [x] **T5.2** Safe deletion: Trash-only + confirmation sheet + multi-select + select-all-but-keeper (F9). DoD met: `trash()` uses `FileManager.trashItem` only (CI grep forbids `removeItem`/`unlink` on user files); confirmation sheet summarizes count + bytes; bulk-action bar multi-select; per-group "Select all but keeper"; per-file failures no longer abort the batch and leave the index consistent (`testTrashContinuesPastAMissingFileAndStaysConsistent`).
- [x] **T5.3** Undo (⌘Z / Edit menu) restoring from Trash. DoD met: Edit ▸ Undo Delete (⌘Z) + toolbar affordance call `undoLastTrash()`, moving files back from the Trash and restoring live results (`testTrashMovesNonKeeperToTrashAndUpdatesState` covers the round-trip). Single-level undo (ponytail).
- [x] **T5.4** Ignore group / not-duplicates persistence (F7/F14). DoD met: "Not duplicates" on each group persists its member pairs (`ignore_pair`) + marks the group ignored, removes it from the live results, and a re-scan that re-finds the same set drops it before it surfaces (`testIgnoredGroupDoesNotRecurOnRescan`). Surfaced groups now carry the store-assigned id (engine emits 0). ponytail: recurrence keyed by `FileRecord.id` (stable while enumeration order is) — content-hash key is the cross-rescan upgrade.

## Milestone 7 — Semantic tier (opt-in)
- [x] **T7.1** `EmbeddingProvider` protocol + deterministic stub. DoD met: stub-driven tests.
- [x] **T7.2** Stage 3 embed-survivors + cosine within buckets + `.semantic` groups + model invalidation (F6). DoD met: the coordinator now runs Stage 3 as an **opt-in scan mode** (`DetectionConfig.deepScan`, off by default so a normal scan never embeds). When on, it embeds **only survivors of the cheaper stages** — one representative per exact/near group plus unmatched docs (golden rule 5), reusing the text already extracted for Stage 2 — buckets vectors and cosines within buckets (no all-pairs), and folds `.semantic` edges into the single clustering pass. Provider is injected + swappable (defaults to `StubEmbeddingProvider`; the app shares one instance with the Model tab, ready for the T7.4 Core ML swap). A whole-stage embed failure is swallowed so the scan never fails because the optional ML tier did. `testDeepScanIsOptInAndFoldsSemanticGroups` (same files: no group fast / one `.semantic` group deep) + `testDeepScanEmbedsOnlySurvivorsNotResolvedFiles` (exact dup collapses to one survivor before embedding) green, alongside the existing F6 stub-acceptance/pruning/model-invalidation tests. **Deviation from the original "`deepScan(sessionID:)` over persisted survivors" sketch:** the store intentionally persists no un-grouped inventory, so deep scan re-derives survivors via the existing enumerate→extract path (a re-runnable scan mode) instead of reading them back — simpler, keeps persistence minimal. ponytail: embedding persistence ("store embeddings with model_id") deferred until a *real* (slow) provider lands in T7.4 — caching a microsecond stub embed is dead weight, and model-invalidation is already proven at the stage level.
- [x] **T7.3** "Deep scan" UI affordance (global, opt-in). DoD met: a "Deep Scan" toolbar action (shown whenever sources exist and no scan is running) triggers a confirmation dialog carrying the **battery note** ("slower and uses more energy — best run while plugged in. Nothing leaves your Mac.") before it runs; confirming calls `runScan(deepScan: true)`. It's an explicit per-run action, never a persisted default, so the expensive tier runs only on demand. The `.embedding` phase reports progress in the existing scan header. ⌘R and the primary Scan button stay the fast path.
- [ ] **T7.4** (Gated) Wire Core ML provider once model pinned by human. DoD: provider swaps without engine changes.

## Milestone 8 — Hardening & release
- [x] **T8.1** Per-file error capture + "Skipped (N)" UI (`ERROR_HANDLING.md`). DoD met: `ScanService` captures `.fileSkipped` events into a `skipped` list (was dropped on the floor); the results view shows a collapsible "Skipped (N)" section with a plain-language reason per `FileIssue.Kind` + Reveal-in-Finder; a corrupt/needs-OCR file never fails the scan (`testSkippedFilesAreCapturedAndScanStillSucceeds`). ponytail: skips are in-memory for the current scan, not persisted into reopened history.
- [x] **T8.2** Performance harness + budgets (`PERFORMANCE.md`). DoD met: `Tools/CorpusGen` generates deterministic synthetic corpora (seeded, known unique/exact/near ratios); `PerfHarnessTests` runs the full cascade over a generated corpus, records wall-clock + throughput + **peak memory** (`getrusage`/`ru_maxrss`) to a CSV, and asserts against budgets on two tiers: `testCascadeThroughputOnSmallCorpus` (fast debug regression smoke on every gate) and `testFullScanBudget` (opt-in `DOPPEL_PERF_BUDGET=1`, `-c release`). **Headline budget verified on Apple Silicon (release):** 50,000 docs scanned through Stage 2 in **14.5 s** (budget < 5 min; 3455 files/sec ≫ 300/sec target) at **304 MB peak** (budget < 1.5 GB). ponytail: budget corpus defaults to ~1.5 KB/doc (count-scaled, not the doc's 200 KB avg); `DOPPEL_PERF_WORDS` bumps byte size for a faithful run. UI-frame/TTI budgets (§1) are UI-layer, tracked under T8.3/manual.
- [~] **T8.3** Accessibility pass (`ACCESSIBILITY.md`). **Code pass done:** decorative SF Symbols hidden; keeper star labeled; `GroupCard` is a combined VoiceOver element reading match-type + confidence + count + reclaimable + explanation as one phrase; skipped rows combine name+reason; ⌘R scan / ⎋ cancel shortcuts added (⌘Z undo, ⌘⌫ trash already present); state never conveyed by color alone (badges have text, keeper has a star). Uses system text styles so Dynamic Type reflows. **Remaining (manual, requires an interactive run I can't drive headless):** the §8 audit checklist — VoiceOver end-to-end, keyboard-only journey, Dynamic Type XL snapshot, Accessibility Inspector zero-errors, Differentiate-Without-Color / Increase-Contrast checks.
- [x] **T8.4** Network-egress CI guard (no file-derived data leaves). DoD met: a CI step (mirroring the deletion guard) statically forbids all networking APIs (URLSession/URLRequest/NW*/CFSocket/`import Network`/`socket(`/`*Task`) in `Packages/*/Sources` + `App`, and asserts the app entitlements request no `network.client`/`server` — proving zero egress code paths (golden rule 1). Sparkle, when integrated, gets its own scoped build config + exclusion. ponytail: static guard over a runtime network-denied sandbox profile — stronger for the no-code guarantee; add the runtime profile as belt-and-suspenders if desired.
- [~] **T8.5** Signing, notarization, Sparkle appcast (`RELEASE.md`). **Code-complete:** Sparkle 2 integrated (package dep in `project.yml`; updater + "Check for Updates…" menu isolated in `App/Doppel/App/Updater.swift`, the sole network touch-point); feed URL + EdDSA public key are Info.plist placeholders; network entitlement is **Release-only** (`Doppel.release.entitlements`) so Debug + the egress guard stay network-free (guard updated to allow only Sparkle's `network.client`, still ban `network.server`); `Scripts/release.sh` does archive→Developer-ID export→notarytool→staple→verify, parameterized on creds. Verified: Release build links Sparkle & bundles `Sparkle.framework`, Info.plist keys land, script syntax-checks. **Remaining (credential-gated, only the maintainer can do):** generate the EdDSA key pair + set the two placeholders, `notarytool store-credentials`, run `release.sh` with a Developer ID cert, and publish the signed appcast — then verify the live update round-trip on a clean machine (RELEASE.md §5).
